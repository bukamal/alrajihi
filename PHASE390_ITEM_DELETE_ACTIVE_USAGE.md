# Phase 390 — Item Delete Active Usage

هذه المرحلة تصحح فهم النظام لرسالة منع حذف المادة.

## المشكلة

حذف الفاتورة في النظام هو حذف منطقي `soft delete`: تبقى أسطر الفاتورة في قاعدة البيانات للأرشفة والتدقيق. لذلك كان فحص حذف المادة يعدّ `invoice_lines` الخام حتى لو كانت الفاتورة الأم محذوفة أو ملغاة. كذلك كانت أوامر الإنتاج الملغاة تظهر كمانع حذف رغم أنها ليست عملية نشطة.

## التصحيح

- فحص حذف المادة أصبح يحسب أسطر الفواتير عبر الفواتير النشطة فقط.
- الفواتير المحذوفة أو الملغاة لم تعد تمنع أرشفة/حذف المادة.
- أوامر الإنتاج الملغاة لم تعد تمنع حذف المادة.
- استهلاكات ومخرجات الإنتاج لا تُحسب إلا إذا كان أمر الإنتاج غير ملغى.
- حركات المخزون المرتبطة بفواتير محذوفة أو أوامر إنتاج ملغاة لم تعد تظهر كحركات نشطة.
- أسطر BOM النشطة لا تزال تمنع الحذف لأنها وصفات تصنيع قائمة، وليست أوامر إنتاج ملغاة.
- رسالة الخطأ أصبحت عربية مفهومة ولا تعرض أسماء الجداول الخام مثل `invoice_line` أو `bom_line`.

## النتيجة

إذا حذفت فواتير البيع/الشراء أو ألغيت أمر إنتاج، لن تبقى هذه السجلات التاريخية وحدها مانعًا لحذف المادة. إذا بقي المنع بسبب BOM، يجب حذف الوصفة أو إزالة المادة منها أولًا.
