# SETTINGS PHASE 147 - Security Events & Diagnostics Governance

## الهدف
تحويل الصلاحيات من سياسة صامتة إلى سياسة قابلة للتتبع والتشخيص.

## المنفذ
- إضافة جدول `security_events` لتسجيل عمليات الرفض الناتجة عن `PermissionService`.
- تسجيل تلقائي عند رفض: الحذف، تعديل الفواتير، تعديل المرتجعات، عرض التقارير، تصدير التقارير.
- إضافة دوال `security_events()` و `denied_events_count()` داخل `PermissionService`.
- إضافة واجهة `security_event_rows()` و `security_denied_count()` داخل `SettingsService`.
- إضافة تبويب جديد داخل الإعدادات: `سجل الصلاحيات`.
- إضافة مؤشر تشخيصي جديد: `عمليات مرفوضة بالصلاحيات`.
- تحديث migrations لإنشاء الجدول والفهارس.

## الأثر العملي
عند تفعيل سياسات المنع من الإعدادات، لن يكتفي النظام بإظهار رسالة رفض؛ بل سيحفظ الحدث في سجل قابل للمراجعة من الإعدادات والتشخيص.

## ملاحظات
- التسجيل غير معطِّل: أي فشل في كتابة سجل الأمان لا يمنع تنفيذ فحص الصلاحية.
- المدير `admin` يبقى مستثنى من المنع وفق سياسة Phase 146.
