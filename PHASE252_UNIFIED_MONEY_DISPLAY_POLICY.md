# PHASE252 — Unified MoneyDisplayPolicy

## الهدف
توحيد عرض العملة والأرقام في الواجهة، الجداول، المجاميع، POS، المطعم، التقارير، والطباعة، مع احترام العملة المعروضة وعدم عرض `float` الخام أو بقايا Decimal مثل `1E-22-`.

## التعديل
- إضافة `alrajhi_client/core/money_display_policy.py` كطبقة عرض مستقلة عن PyQt.
- ربط `currency.CurrencyManager.format_amount()` بالسياسة الجديدة دون تحويل عملة داخل التنسيق.
- ربط `utils.format_currency()` بالسياسة الجديدة للتوافق مع الكود القديم.
- ربط جداول الفواتير والمرتجعات `TransactionLineModel` بالسياسة الجديدة.
- ربط لوحة المجاميع `TransactionTotalsPanel` بالسياسة الجديدة.
- ربط POS و Restaurant order models بالسياسة الجديدة.
- ربط `print_templates.py` بالسياسة الجديدة مع إبقاء fallback داخلي آمن.
- تحسين هامش الربح في محرر المواد ليعرض العملة عبر السياسة نفسها.

## قاعدة مهمة
السياسة الجديدة **لا تقوم بتحويل العملة**. هي تنسّق فقط المبالغ التي وصلت إلى الواجهة بعملة العرض. التحويل يبقى في `CurrencyManager.convert()/to_display()/from_display()` أو طبقة الخدمة المناسبة.

## أثر الشبكة و API
هذه المرحلة لا تغير تخزين القيم ولا REST payload. هي توحد طبقة العرض فقط، لذلك لا تكسر العميل/السيرفر ولا تعدد المستخدمين. المستند يستطيع فرض عملته عبر `currency` أو `currency_code` أو `display_currency` داخل payload.

## الاختبارات
أضيف اختبار `tests/test_phase252_unified_money_display_policy.py` للتحقق من:
- عدم ظهور `1E-22-`.
- تقريب بقايا float مثل `14999.999999999999` إلى `15,000.00`.
- احترام عملة المستند `SYP` حتى لو إعداد العرض العام `USD`.
- فصل تنسيق الكميات عن الأموال.
- حراسة static لأهم أسطح العرض.
