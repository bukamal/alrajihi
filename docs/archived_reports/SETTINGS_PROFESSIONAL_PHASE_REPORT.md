# SETTINGS PROFESSIONALIZATION PHASE REPORT

## الهدف
تحويل وحدة الإعدادات من شاشة بسيطة إلى مركز إعدادات ERP قابل للتوسع دون كسر البنية الحالية.

## ما تم تطبيقه

### 1) بنية الإعدادات
- توسيع جدول `settings` ليقبل:
  - `key`
  - `value`
  - `category`
  - `updated_at`
- إضافة ترقيات آمنة لقواعد البيانات القديمة بإضافة الأعمدة عند غيابها.
- تحديث حفظ الإعدادات المحلي والخادمي لإضافة `category` و`updated_at` تلقائيًا مع fallback للتوافق القديم.

### 2) إعدادات الشركة
- إضافة:
  - السجل التجاري
  - موقع الويب
- حفظها ضمن `QSettings` مع الحقول الحالية.

### 3) تبويبات إعدادات جديدة
تمت إضافة تبويبات احترافية داخل `SettingsWidget`:
- الفواتير
- الوحدات
- المرتجعات
- المخزون
- التصنيع
- التقارير
- التشخيص

### 4) إعدادات واجهة المستخدم
تم توسيع تبويب المظهر ليشمل:
- حجم الخط
- حجم الصفوف
- إظهار/إخفاء البحث العالمي
- الصفحة الافتراضية عند التشغيل
- تذكر التبويب الأخير

### 5) صفحة تشخيص النظام
تمت إضافة صفحة تعرض:
- وضع الاتصال
- مصدر البيانات
- مسار قاعدة البيانات
- حجم قاعدة البيانات
- إحصاءات الجداول الأساسية
- حالة الجداول الأساسية
- فحص مخزون سالب
- فحص أسطر فواتير يتيمة
- `PRAGMA quick_check`

## ملاحظات مهمة
- الإعدادات تم حفظها الآن كمفاتيح مركزية عبر `SettingsService`، لكنها لم تُربط كلها بعد بمنطق الفواتير/المخزون/التصنيع. هذا الربط يجب أن يتم تدريجيًا داخل كل وحدة.
- لم تتم إضافة LIFO عمدًا؛ تم الإبقاء على `AVERAGE` و `FIFO` لأنها أنسب عمليًا.
- الاختبار المتاح في هذه البيئة اقتصر على فحص الصياغة `compileall` بسبب عدم توفر PyQt5 Runtime.

## المرحلة التالية المقترحة
1. ربط `invoice/sales_prefix` و `invoice/purchase_prefix` بمولد أرقام الفواتير.
2. ربط `inventory/allow_negative_stock` بمنطق البيع والتصنيع.
3. ربط `returns/max_days` و `returns/prevent_quantity_exceed` بمنطق المرتجعات.
4. جعل إعدادات الطباعة والتقارير تقرأ من المفاتيح الجديدة حيث يلزم.
