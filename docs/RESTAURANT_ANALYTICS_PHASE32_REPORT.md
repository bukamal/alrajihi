# Restaurant Analytics Phase 32

Implemented a read-only Restaurant Analytics layer.

## Added
- `restaurant_analytics()` in local/server repository boundaries.
- Remote/local gateway and service contracts.
- `GET /api/restaurant/analytics`.
- Touch-friendly analytics panel in the restaurant dashboard.
- Arabic, English, and German translation keys.
- Tests for KPI calculation and route-boundary cleanliness.

## KPIs
- Open sessions and table statuses.
- Payment totals by method.
- Top menu items.
- Table usage.
- Waiter performance.
- Kitchen station load.
