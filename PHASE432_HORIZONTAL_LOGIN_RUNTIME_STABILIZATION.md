# Phase 432 — Horizontal Login Runtime Stabilization

## هدف المرحلة

تثبيت واجهة تسجيل الدخول العرضية بعد Phase 431 بناءً على لقطة Runtime فعلية أظهرت أن التصميم أصبح أفقيًا، لكنه بقي مضغوطًا، مع أزرار عنوان كبيرة على شكل تبويبات، وتداخل في خيارات اللغة/تذكر المستخدم، ورسالة خطأ فارغة تظهر كشريط أحمر، ونصوص غير مقروءة داخل لوحة الهوية.

## ما تم تثبيته

- رفع أبعاد نافذة الدخول الافتراضية إلى `1120 x 660`.
- رفع الحد الأدنى إلى `980 x 600` لمنع الرجوع للشكل الضيق.
- تثبيت عرض لوحة الهوية إلى `390px` وعرض نموذج الدخول إلى `610px`.
- إضافة `login_horizontal_panel_min_height` لضمان أن لوحة الهوية ونموذج الدخول لا ينضغطان عموديًا.
- تحويل شريط العنوان الموروث إلى شريط صغير نظيف `LoginRuntimeTitleBar` بارتفاع 40px.
- تصغير أزرار التصغير والإغلاق إلى 30px ومنعها من الظهور كبطاقات/تبويبات عائمة.
- إخفاء أيقونة العنوان غير اللازمة داخل تسجيل الدخول.
- جعل رسائل الخطأ/النجاح عبر `loginRuntimeMessage` مع حالة `messageState` بدل `QLabel#danger` الفارغ.
- حجز ارتفاع محسوب للتحذير ورسالة الخطأ حتى لا تدفع الحقول أو تقصّها.
- جعل نصوص لوحة الهوية شفافة الخلفية لمنع الشرائط البيضاء غير المقروءة.
- عدم لمس منطق التفعيل، المستخدمين، الجلسة، اللغة، أو المصادقة.
