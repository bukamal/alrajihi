# Phase 331 — UI Registry and Shell Manifest

هذه المرحلة هي نقطة البداية لتوحيد المشروع كاملًا بدل الاستمرار بتعديلات موضعية لكل شاشة.

## ما تم تأسيسه

- إنشاء سجل مركزي لكل واجهة رئيسية في `alrajhi_client/workspace/registry/ui_manifest.py`.
- نقل metadata الأساسية للصفحات إلى السجل: `page_id`, `title_key`, `group_key`, `nav_group`, `icon`, `workspace_type`.
- تعريف سياسة ظهور شريط الإجراءات من السجل، بحيث لوحة التحكم لا تعرض شريط الإجراءات بينما واجهات التشغيل والمستندات تعرضه.
- تعريف جداول كل واجهة مبدئيًا عبر `WorkspaceTableSpec` استعدادًا لعقد الأعمدة الموحد في Phase 334.
- ربط `module_visibility_policy` بالسجل بدل تكرار مفاتيح تفعيل الصفحات في أكثر من مكان.
- جعل `MainWindow` يقرأ `PAGE_META_KEYS`, `NAV_GROUP_BY_PAGE`, وترتيب إنشاء الصفحات من السجل.

## نقطة النهاية المضافة للباركود

تم تعريف `WorkspaceBarcodeProfileSpec` كنقطة نهاية ملزمة لنظام طباعة الباركود الموحد لاحقًا.

الملفات الشخصية المعرّفة:

- `items.default` لطباعة باركود المواد.
- `apparel.variant_labels` لطباعة باركود متغيرات الألبسة.
- `restaurant.menu_items` لطباعة باركود أصناف المطعم.
- `restaurant.table_labels` لطباعة QR/باركود الطاولات.
- `cafe.products` لطباعة باركود منتجات الكافي.
- `cafe.modifier_labels` لطباعة باركود إضافات الكافي.

كل profile يدعم:

- الطباعة المفردة.
- الطباعة المتعددة.
- إعدادات مستقلة تبدأ من `printing/barcode/...`.
- الالتزام بمسار Browser HTML فقط.

## لماذا هذه المرحلة مهمة

هذه المرحلة لا تغيّر شكل كل الواجهات دفعة واحدة. هي تمنع فقدان أي واجهة أو جدول أو مسار عند تنفيذ المراحل التالية:

- توحيد الخطوط والأحجام.
- ترقية شريط القوائم.
- توحيد أشرطة الإجراءات.
- توحيد إدارة الأعمدة.
- ربط الأعمدة بالطباعة والتصدير.
- ربط طباعة الباركود الخاصة بالمواد والمطعم والكافي والألبسة بالإعدادات.
