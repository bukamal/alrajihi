# Phase 282 — Report Table Calculation / Module Visibility / Dashboard Cleanup

## الهدف
إصلاح جداول التقارير التي كانت تظهر فارغة أو تعتمد على حقول رصيد ثابتة، وربط تفعيل/تعطيل الوحدات في الإعدادات بشريط القوائم و Quick Open، وحذف صف بطاقات لوحة التحكم والرسم البياني العلوي نهائيًا.

## نطاق التقارير
- قائمة الدخل وصافي الربح.
- الأستاذ العام وميزان المراجعة الكامل عبر `journal_lines.journal_entry_id` و `journal_entries`.
- كشف حساب عميل ومورد مع ملخص افتراضي عند عدم اختيار طرف.
- أرصدة العملاء والموردين محسوبة من الفواتير، المرتجعات، والسندات بدل الاعتماد على `balance` فقط.
- أعمار ديون العملاء والموردين من الرصيد المحسوب وآخر حركة.
- حركة المادة، المواد الراكدة، المواد تحت حد الطلب، وأرباح الفواتير.
- مطابقة ledger والقراءة المزدوجة مع مفاتيح `difference_quantity`, `operational_quantity`, `ledger_quantity`.
- أوامر التصنيع وكلفة المنتج.

## نطاق الواجهة
- إضافة `workspace.navigation.module_visibility_policy` كمصدر واحد لإظهار/إخفاء الوحدات.
- عند إلغاء تفعيل المطعم أو التصنيع أو التقارير أو POS أو المالية أو المخزون أو الأطراف أو الفروع أو المستخدمين، تختفي من شريط القوائم و Quick Open، ويمنع فتحها مباشرة.
- حفظ إعدادات العقود الموحدة يعيد بناء القوائم فورًا وينقل المستخدم إلى لوحة التحكم إذا كانت الصفحة الحالية أصبحت معطلة.

## لوحة التحكم
تم إلغاء بناء صف KPI العلوي ولوحة الرسم البياني، مع إبقاء باقي اللوحة: الاختصارات، معلومات الشركة، التنبيهات، وبيانات المشروع.
