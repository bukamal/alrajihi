# Phase 38 — Dashboard & Navigation Redesign Foundation

## الهدف
إعادة تصميم لوحة التحكم الرئيسية بأسلوب تشغيلي قريب من واجهات ERP الحديثة مثل البسيط: ملخص سريع، اختصارات يومية، تنبيهات، آخر الحركات، ومؤشرات مراقبة التشغيل.

## التعديلات

- إعادة بناء `DashboardWidget` باستخدام Scroll Area حتى تناسب الشاشات المختلفة.
- إضافة Hero Panel علوي يحتوي:
  - عنوان لوحة التحكم التشغيلية.
  - أزرار تحديث/مراقبة/تقارير.
  - حالة API.
  - حالة Offline Queue.
  - حالة Ledger.
- تحويل المؤشرات إلى بطاقات واضحة:
  - مبيعات اليوم/الفترة.
  - مشتريات اليوم/الفترة.
  - النقدية.
  - صافي الربح.
  - ذمم العملاء.
  - ذمم الموردين.
  - المصروفات.
  - حالة التشغيل.
- إضافة لوحة اختصارات يومية:
  - POS.
  - فاتورة بيع.
  - فاتورة شراء.
  - عميل جديد.
  - مورد جديد.
  - مادة جديدة.
  - سند قبض.
  - سند دفع.
  - مراقبة التشغيل.
- إضافة مركز تنبيهات موسّع.
- إضافة جدول آخر الحركات.
- إضافة لوحة حالة تشغيل مختصرة.
- إصلاح مفاتيح فتح الفواتير من لوحة التحكم:
  - sale -> `sales_invoices`
  - purchase -> `purchase_invoices`

## خصائص السلامة

- التعديل UI-only تقريباً.
- لا يوجد تغيير في قواعد البيانات.
- لا يوجد تغيير في منطق الفواتير أو المخزون أو Ledger.
- عند فشل مراقبة التشغيل، لا تنهار لوحة التحكم؛ تعرض قيم افتراضية.
