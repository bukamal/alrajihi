# Phase 397 — Feature Activation Gate

## الهدف
توحيد طلب مفتاح التفعيل للوحدات الاختيارية بنفس أسلوب تفعيل الشبكة.

## النطاق
- التصنيع.
- واجهة المطعم.
- واجهة المقهى.
- واجهة الألبسة.
- تفعيل الشبكة بقي متوافقًا مع الملف القديم `network_license.dat`.

## التنفيذ
- إضافة خدمة تفعيل عامة `activate_feature()` و `check_feature_activation()`.
- إضافة ملف ترخيص مستقل لكل وحدة داخل مجلد بيانات المستخدم.
- إضافة `ModuleActivationDialog` كواجهة موحدة لإدخال مفتاح التفعيل.
- حماية `switch_page()` قبل فتح الوحدات المدفوعة.
- حماية اختصارات مستندات التصنيع المباشرة مثل الوصفة وأمر الإنتاج.
- ربط تفعيل الشبكة بنفس الحوار الموحد.

## السلوك
عند الدخول إلى وحدة غير مفعلة، يظهر مربع تفعيل. عند نجاح التفعيل تُفتح الواجهة. عند الإلغاء أو فشل المفتاح لا يتم فتح الواجهة.
