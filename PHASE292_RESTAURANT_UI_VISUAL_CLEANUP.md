# Phase 292 — Restaurant UI Visual Cleanup

## الهدف
تنظيف واجهات المطعم الظاهرة في التشغيل اليومي دون إضافة دورة عمل جديدة: الطلب الحالي، شاشة المطبخ، ومناطق الإجراءات المرئية.

## النطاق
- إعادة تنظيم رأس الطلب الحالي داخل بطاقة واضحة.
- تحويل الملخص المالي من سطر تقني مزدحم إلى بطاقات مالية منفصلة.
- إزالة ظهور مفاتيح تقنية مثل `restaurant.subtotal` عبر استكمال ترجمات المستخدم.
- تجميع أزرار الطلب في مجموعات تشغيلية: الطلب، المطبخ، الدفع والإغلاق.
- تحسين شاشة المطبخ بصريًا عبر جسم لوحة واضح وعناوين تذاكر أكثر قابلية للقراءة.
- إضافة تنسيقات QSS جديدة لعناصر المطعم المرئية.

## الملفات الأساسية
- `alrajhi_client/views/restaurant/restaurant_pos_widget.py`
- `alrajhi_client/views/restaurant/kitchen_display_widget.py`
- `alrajhi_client/i18n/translator.py`
- `alrajhi_client/theme/qss.py`
- `tests/test_phase292_restaurant_ui_visual_cleanup.py`

## ملاحظات
هذه المرحلة لا تغيّر سياسة الدفع أو دورة الطلب أو منطق خصم المخزون. هي مرحلة UX مرئية فقط مبنية فوق Phase 287–291.
